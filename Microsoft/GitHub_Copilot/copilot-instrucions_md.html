1. はじめに

GitHub Copilot を使用している皆さんは .github/copilot-instructions.md というファイルをご存じでしょうか。

このファイルにリポジトリ用のカスタム命令やルールを記載しておくことで、あなたは GitHub Copilot の AI 支援を効率よく受けることができるようになります。

GitHub オリジナルの記事は、下記を参照してください。

GitHub Copilot のリポジトリ カスタム命令を追加する - GitHub Docs

 

2. copilot-instructions.md の紹介

"copilot-instructions.md" は、対象リポジトリにおける GitHub Copilot への指示書です。このファイルは、GitHub Copilot がユーザーの指示を処理する前に真っ先に読み込む指示書として機能します。特別な設定は不要で、プロジェクトの .github フォルダに copilot-instructions.md という名前で置くだけで認識されます。 Copilot は、この指示書を読んでからユーザーの指示に応えるため、この "copilot-instructions.md" を作りこむほどプロジェクトの期待に沿った理想の回答を得やすくなります。

# C++ リポジトリにおける手順

## Overview
このリポジトリには、最新の C++ ガイドラインに準拠した C++ アプリケーションが含まれています。
コードはクリーンで効率的で、十分に文書化されている必要があります。 

## コーディングスタイル
- **命名規則:**
  - Classe と struct は `PascalCase` を使用する.
  - 関数と変数は `snake_case` を使用する.
  - グローバル定数は 'SCREAMING_SNAKE_CASE' を使用する.
- **インデント:** インデントにスペース4文字を使用する. tab を使用してはいけない.
- **括弧:** "Allman style" の括弧を使用 (各括弧は新しい行に記載).
- **空白:**
  - バイナリ操作 (`+`, `-`, `*`, `/`, `=`) の後にスペースを加える.
  - `if`, `for`, `while` などのキーワード後にスペースを加えること.
- **ヘッダ:**
  - 必ずインクルードガードを使用すること.
  - include ディレクティブをアルファベット順でソートする.

## 言語と機能
- **Modern C++:** Prefer modern C++17/20 features (e.g., `std::optional`, `std::string_view`, structured bindings).
- **Resource Management:** Use RAII (Resource Acquisition Is Initialization) and smart pointers (`std::unique_ptr`, `std::shared_ptr`) to manage resources.
- **Const Correctness:** Use `const` where appropriate for variables, function parameters, and member functions to improve code safety.
- **Error Handling:** Use exceptions for exceptional errors. For expected errors, prefer return values (e.g., `std::optional`, `std::error_code`).
- **Standard Library:** Prefer the standard library over third-party libraries unless explicitly specified.

## 文書化とコメント
- **Doxygen:** Doxygen-style のコメントを全ての public classes, functions, そして複雑なロジックへ適用する.
- **関数ヘッダ:** 全ての public 関数は、その関数の概要 (brief description) およびパラメータ、戻り値について記載する.
- **Inline Comments:** Use inline comments sparingly to explain non-obvious code.

## 例

### Class Example
```cpp
class MyClass {
public:
    /**
     * @brief MyClass オブジェクトのコンストラクタ
     * @param name オブジェクトの名前
     */
    MyClass(std::string_view name);
    
    /**
     * @brief オブジェクトの名前を取得
     * @return オブジェクトの名前を string 形式で
     */
    [[nodiscard]] std::string_view get_name() const;

private:
    std::string name_;
};

# Instructions for C++ Repository

## Overview
This repository contains a C++ application that follows modern C++ guidelines. The code should be clean, efficient, and well-documented.

## Coding Style
- **Naming Conventions:**
  - Classes and structs should use `PascalCase`.
  - Functions and variables should use `snake_case`.
  - Global constants should use `SCREAMING_SNAKE_CASE`.
- **Indentation:** Use 4 spaces for indentation. Do not use tabs.
- **Braces:** Use Allman style braces (each brace on a new line).
- **Whitespace:**
  - Add a space before and after binary operators (`+`, `-`, `*`, `/`, `=`).
  - Add a space after keywords like `if`, `for`, `while`.
- **Headers:**
  - Always use include guards.
  - Sort include directives alphabetically.

## Language and Features
- **Modern C++:** Prefer modern C++17/20 features (e.g., `std::optional`, `std::string_view`, structured bindings).
- **Resource Management:** Use RAII (Resource Acquisition Is Initialization) and smart pointers (`std::unique_ptr`, `std::shared_ptr`) to manage resources.
- **Const Correctness:** Use `const` where appropriate for variables, function parameters, and member functions to improve code safety.
- **Error Handling:** Use exceptions for exceptional errors. For expected errors, prefer return values (e.g., `std::optional`, `std::error_code`).
- **Standard Library:** Prefer the standard library over third-party libraries unless explicitly specified.

## Documentation and Comments
- **Doxygen:** Use Doxygen-style comments for all public classes, functions, and complex logic.
- **Function Headers:** Every public function should have a brief description of what it does, its parameters, and its return value.
- **Inline Comments:** Use inline comments sparingly to explain non-obvious code.

## Examples

### Class Example
```cpp
class MyClass {
public:
    /**
     * @brief Constructs a new MyClass object.
     * @param name The name of the object.
     */
    MyClass(std::string_view name);
    
    /**
     * @brief Gets the name of the object.
     * @return The name as a string view.
     */
    [[nodiscard]] std::string_view get_name() const;

private:
    std::string name_;
};
